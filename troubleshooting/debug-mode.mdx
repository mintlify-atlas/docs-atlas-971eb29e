Debug mode provides detailed information about your system and auto-cpufreq's operation, which is essential for troubleshooting issues.

## Using the --debug Flag

The `--debug` flag outputs comprehensive system information that helps diagnose problems with auto-cpufreq.

### Basic Usage

```bash
auto-cpufreq --debug
```

This command displays:

- System information (OS, kernel, architecture)
- CPU information (model, cores, threads)
- Current CPU frequency driver and governor
- Available CPU governors
- Current CPU frequencies (per core)
- CPU usage statistics (per core)
- CPU temperature readings (per core)
- Battery state and power supply information
- Current auto-cpufreq configuration
- Turbo boost status
- System load averages

### Saving Debug Output

For bug reports, save the debug output to a file:

```bash
auto-cpufreq --debug > auto-cpufreq-debug.txt 2>&1
```

This creates a text file with all debug information that you can attach to issue reports.

## What Debug Output Shows

Debug mode provides insights into various system components:

### CPU Driver Information

```
CPU Driver: intel_pstate
Available Governors: performance powersave
Current Governor: powersave
```

This helps identify if you're using the correct driver for auto-cpufreq (ideally `acpi-cpufreq`).

<Note>
If you see `intel_pstate` or `amd-pstate`, consider switching to `acpi-cpufreq`. See [Driver Conflicts](/troubleshooting/driver-conflicts) for instructions.
</Note>

### Frequency Scaling

```
CPU Frequency Scaling:
Core 0: 1200 MHz
Core 1: 1400 MHz
Core 2: 1200 MHz
Core 3: 1350 MHz
```

This shows current operating frequencies and helps identify if scaling is working properly.

### Temperature Monitoring

```
CPU Temperature:
Average: 55°C
Core 0: 54°C
Core 1: 56°C
Core 2: 55°C
Core 3: 57°C
```

Helps diagnose overheating issues.

### Battery State

```
Battery State: Discharging
Battery Percentage: 67%
Power Supply: BAT0
```

Shows if auto-cpufreq is correctly detecting power state for profile switching.

### Turbo Boost Status

```
Turbo Boost: Enabled
```

Indicates whether turbo mode is currently active.

## Debug Mode vs. Stats Mode

It's important to understand the difference between debug mode and stats mode:

| Mode | Command | Purpose |
|------|---------|----------|
| **Debug** | `auto-cpufreq --debug` | Shows detailed system information and configuration for troubleshooting |
| **Stats** | `auto-cpufreq --stats` | Shows live statistics of daemon operations (only works when daemon is installed) |

<Info>
Use `--debug` for troubleshooting. Use `--stats` to monitor the daemon's ongoing operations.
</Info>

## Using Stats Mode

If the daemon is installed and running, use stats mode to see live optimization data:

```bash
auto-cpufreq --stats
```

Stats mode shows:
- Real-time frequency changes
- Governor switches
- Turbo mode toggles
- Power state transitions
- Historical optimization data

7. **Installation method** (installer, Snap, AUR, etc.)

## Creating Detailed Bug Reports

### Gather All Relevant Information

1. **Run debug mode**:
   ```bash
   auto-cpufreq --debug > debug-output.txt 2>&1
   ```

2. **Check daemon status** (if installed):
   ```bash
   systemctl status auto-cpufreq > daemon-status.txt 2>&1
   ```
   
   For Snap:
   ```bash
   systemctl status snap.auto-cpufreq.service.service > daemon-status.txt 2>&1
   ```

3. **Include your configuration** (if using custom config):
   ```bash
   cat /etc/auto-cpufreq.conf > my-config.txt
   ```

4. **Check for conflicting services**:
   ```bash
   systemctl status tlp power-profiles-daemon tuned > conflicts-check.txt 2>&1
   ```

### Attach Files to Issue

When creating the GitHub issue, attach:
- `debug-output.txt`
- `daemon-status.txt` (if daemon is installed)
- `my-config.txt` (if using custom configuration)
- `conflicts-check.txt` (to rule out conflicts)

## Common Debug Patterns

Certain patterns in debug output indicate specific issues:

### Driver Issues

If you see:
```
CPU Driver: intel_pstate
```
or
```
CPU Driver: amd-pstate
```

And experiencing problems → See [Driver Conflicts](/troubleshooting/driver-conflicts)

### Conflicting Services

If debug output shows multiple power management services active → See [Common Issues](/troubleshooting/common-issues#checking-for-conflicting-services)

### Temperature Problems

If temperatures are consistently high → Check:
- Thermal paste application
- Dust buildup
- Conflicting services
- Consider installing thermald

### Scaling Issues

If all cores show the same frequency and aren't changing → Check:
- CPU driver (should be `acpi-cpufreq`)
- Daemon is running
- No conflicting services

## Advanced Debugging

For advanced users who need more detailed logging:

### Check System Logs

```bash
# View auto-cpufreq daemon logs
journalctl -u auto-cpufreq -n 100 --no-pager

# Follow logs in real-time
journalctl -u auto-cpufreq -f
```

For Snap installations:
```bash
journalctl -u snap.auto-cpufreq.service.service -n 100 --no-pager
```

### Monitor in Real-Time

Combine multiple tools for comprehensive monitoring:

```bash
# Terminal 1: Watch stats
watch -n 1 auto-cpufreq --stats

# Terminal 2: Monitor temperatures
watch -n 1 sensors

# Terminal 3: Watch CPU frequencies
watch -n 1 "grep MHz /proc/cpuinfo"
```
