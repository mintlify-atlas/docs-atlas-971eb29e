# Debug Mode

The `--debug` command displays comprehensive diagnostic information about your system, CPU state, and auto-cpufreq configuration for troubleshooting purposes.

## Usage

```bash
sudo auto-cpufreq --debug
```

<Note>
Debug mode requires root privileges to access all system information.
</Note>

## What It Shows

Debug mode displays:

- **Distribution information** - OS name, version, kernel
- **System information** - CPU model, cores, architecture  
- **Python information** - Python version, psutil version
- **Device information** - Hostname, uptime
- **Battery information** - State, percentage, charging thresholds
- **CPU frequency** - Current frequencies per core
- **CPU usage** - Per-core utilization
- **CPU temperature** - Per-core temperatures
- **Load average** - System load (1, 5, 15 minutes)
- **Current governor** - Active CPU governor
- **Available governors** - All available governors
- **CPU driver** - Active CPU frequency driver
- **Turbo boost status** - Current turbo state
- **Scaling information** - Min/max frequencies
- **App resource usage** - auto-cpufreq memory and CPU usage
- **Configuration file** - Active config (if any)

## Understanding Debug Output

### Distribution Info

Shows your Linux distribution and kernel version. Useful for identifying distribution-specific issues.

### System Info  

Displays CPU model, core count, and architecture. Helps verify hardware detection.

### App Version

Shows installed auto-cpufreq version. Important for knowing if you're on the latest release.

### Python Info

Lists Python version and package versions. Helps diagnose dependency issues.

### Battery Info

- **Charging state** - AC vs battery power
- **Battery percentage** - Current charge level
- **Thresholds** - If battery charging thresholds are configured

### CPU Information

Shows current frequencies, usage, and temperatures. Use this to verify:
- Frequencies are scaling correctly
- Temperature sensors are detected
- CPU usage is being monitored

### Governor & Driver

Displays:
- Active governor (performance, powersave, etc.)
- Available governors on your system
- CPU frequency driver (intel_pstate, acpi-cpufreq, amd-pstate)

### Turbo Boost

Shows whether turbo is currently enabled or disabled.

### Scaling Info

Displays frequency limits:
- Min frequency - Lowest CPU can go
- Max frequency - Highest CPU can reach
- Driver - How frequency is controlled

### App Resource Usage

Shows auto-cpufreq's own resource consumption. Should be minimal (< 50 MB RAM, < 2% CPU).

## Additional Debug Information

For more detailed debugging:

### Check CPU Driver

```bash
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_driver
```

### Check Available Governors

```bash
cat /sys/devices/system/cpu/cpu0/cpufreq/scaling_available_governors
```

### Check Turbo Boost (Intel)

```bash
cat /sys/devices/system/cpu/intel_pstate/no_turbo
cat /sys/devices/system/cpu/intel_pstate/turbo_pct
```

### Check Turbo Boost (AMD)

```bash
cat /sys/devices/system/cpu/cpufreq/boost
```

### Check Temperature Sensors

```bash
sensors
```

### Check Battery Thresholds

```bash
cat /sys/class/power_supply/BAT0/charge_start_threshold
cat /sys/class/power_supply/BAT0/charge_stop_threshold
```

## Daemon Logs

For daemon-specific issues, check systemd logs:

```bash
# View recent logs
sudo journalctl -u auto-cpufreq -n 100

# Follow logs in real-time
sudo journalctl -u auto-cpufreq -f

# Logs since boot
sudo journalctl -u auto-cpufreq -b

# Logs with timestamps
sudo journalctl -u auto-cpufreq --since "1 hour ago"
```

## Common Issues Revealed by Debug

<AccordionGroup>
  <Accordion title="Wrong CPU driver detected">
    If debug shows `intel_pstate` but you want `acpi-cpufreq`:
    ```bash
    # Add to GRUB_CMDLINE_LINUX_DEFAULT in /etc/default/grub
    intel_pstate=disable
    
    sudo update-grub
    sudo reboot
    ```
  </Accordion>
  
  <Accordion title="No governors available">
    Missing governors usually means:
    - CPU driver not loaded
    - Incompatible CPU
    - Kernel module missing
    
    Check loaded modules:
    ```bash
    lsmod | grep cpufreq
    ```
  </Accordion>
  
  <Accordion title="Temperature not detected">
    Install and configure sensors:
    ```bash
    sudo apt install lm-sensors
    sudo sensors-detect
    sensors
    ```
  </Accordion>
  
  <Accordion title="Battery thresholds not working">
    Check if your device is supported:
    ```bash
    ls /sys/class/power_supply/BAT0/charge_*_threshold
    ```
    
    If files don't exist, your device doesn't support thresholds.
  </Accordion>
</AccordionGroup>

## Exporting Debug Output

Save debug output for sharing:

```bash
# Save to file
sudo auto-cpufreq --debug > ~/auto-cpufreq-debug.txt 2>&1

# With timestamp
sudo auto-cpufreq --debug > ~/debug-$(date +%Y%m%d-%H%M%S).txt 2>&1

# Include system info
{
  echo "=== auto-cpufreq debug ==="
  sudo auto-cpufreq --debug
  echo ""
  echo "=== Daemon status ==="
  systemctl status auto-cpufreq
  echo ""
  echo "=== Daemon logs ==="  
  sudo journalctl -u auto-cpufreq -n 50 --no-pager
} > ~/complete-debug.txt 2>&1
```
