# Monitor Mode

The `--monitor` command allows you to preview what auto-cpufreq would do on your system without making any actual changes.

## Usage

```bash
sudo auto-cpufreq --monitor
```

<Note>
Monitor mode requires root privileges to read CPU information but makes no changes to your system.
</Note>

## What It Does

Monitor mode displays:

- **System information** - OS, kernel, distribution
- **CPU information** - Model, architecture, core count
- **CPU frequency** - Current frequency per core and system average
- **CPU usage** - Per-core and system-wide utilization
- **CPU temperature** - Per-core and average temperatures
- **Battery state** - Charging status and percentage
- **Governor information** - Available and current governors
- **Scaling information** - Current CPU driver and frequency range
- **Turbo boost status** - Whether turbo is enabled or disabled
- **Optimization suggestions** - What auto-cpufreq would do in daemon mode

## Using with Custom Config

You can test monitor mode with a custom configuration file:

```bash
sudo auto-cpufreq --monitor --config /path/to/config.conf
```

This will show suggestions based on your custom configuration.

## Exiting Monitor Mode

Monitor mode runs continuously and updates the display. To exit:

- Press `Ctrl+C` to stop monitoring

<Info>
Monitor mode does not make any changes to your system, so you can exit at any time without affecting CPU settings.
</Info>
