## Command Categories

### Monitoring & Testing

Commands for viewing system state and testing optimizations without permanent changes:

- `--monitor` - Preview optimization suggestions
- `--live` - Apply temporary optimizations
- `--stats` - View daemon statistics
- `--debug` - Show debug information

### Daemon Management

Commands for installing, updating, and removing the permanent daemon:

- `--install` - Install the daemon
- `--remove` - Remove the daemon
- `--update` - Update to latest version

### Configuration & Override

Commands for customizing behavior:

- `--config` - Use custom config file
- `--force` - Override governor
- `--turbo` - Override turbo mode

### Utilities

Additional helpful commands:

- `--bluetooth_boot_off` / `--bluetooth_boot_on` - Bluetooth control
- `--version` - Show version
- `--help` - Display help

## Common Usage Patterns

<AccordionGroup>
  <Accordion title="First-time setup">
    ```bash
    # 1. Preview what auto-cpufreq will do
    sudo auto-cpufreq --monitor
    
    # 2. Test with temporary changes
    sudo auto-cpufreq --live
    
    # 3. Install for permanent use
    sudo auto-cpufreq --install
    
    # 4. View statistics
    auto-cpufreq --stats
    ```
  </Accordion>
  
  <Accordion title="Using custom configuration">
    ```bash
    # Create config file at /etc/auto-cpufreq.conf
    # Then install daemon (it will auto-detect the config)
    sudo auto-cpufreq --install
    
    # Or specify custom config location
    sudo auto-cpufreq --config /path/to/config.conf --monitor
    ```
  </Accordion>
  
  <Accordion title="Temporary performance boost">
    ```bash
    # Force performance mode
    sudo auto-cpufreq --force=performance
    
    # Force turbo always on
    sudo auto-cpufreq --turbo=always
    
    # Reset to normal
    sudo auto-cpufreq --force=reset
    sudo auto-cpufreq --turbo=auto
    ```
  </Accordion>
  
  <Accordion title="Troubleshooting">
    ```bash
    # Get debug information
    sudo auto-cpufreq --debug
    
    # Check daemon status
    systemctl status auto-cpufreq
    
    # View daemon logs
    sudo journalctl -u auto-cpufreq -f
    ```
  </Accordion>
</AccordionGroup>

## Root Privileges

<Warning>
Most auto-cpufreq commands require root privileges because they modify system CPU settings. Use `sudo` for commands that make changes.
</Warning>

Commands that require `sudo`:
- `--monitor`
- `--live`
- `--install`
- `--remove`
- `--update`
- `--force`
- `--turbo`
- `--bluetooth_boot_off` / `--bluetooth_boot_on`
- `--debug`

Commands that work without `sudo`:
- `--stats` (if daemon is installed)
- `--version`
- `--help`
