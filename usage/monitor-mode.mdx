## What is Monitor Mode?

Monitor mode allows you to observe what auto-cpufreq would do to optimize your system without actually making any changes. This is a read-only mode that provides suggestions and displays real-time system information, making it perfect for evaluating auto-cpufreq before committing to permanent changes.

<Info>
  No changes are made to your system in monitor mode. This is solely to demonstrate what auto-cpufreq could do for your system.
</Info>

## Running Monitor Mode

To start monitor mode, run the following command with sudo privileges:

```bash
sudo auto-cpufreq --monitor
```

<Note>
  Root privileges are required because auto-cpufreq needs to read system-level CPU and power management information.
</Note>

## What Information Does Monitor Mode Show?

Monitor mode displays a comprehensive real-time view of your system, split into two columns:

### Left Column - System Information & CPU Stats

- **System Information**
  - Linux distribution name and version
  - Linux kernel version
  - Processor model
  - Number of cores
  - System architecture
  - CPU driver in use

- **Current CPU Stats**
  - CPU maximum frequency
  - CPU minimum frequency
  - Per-core information:
    - Usage percentage
    - Temperature (°C)
    - Current frequency (MHz)
  - CPU fan speed (if available)

### Right Column - Battery, Scaling & System Stats

- **Battery Stats** (if applicable)
  - Battery status (charging/discharging)
  - Battery percentage
  - AC power status
  - Charging thresholds (if configured)

- **CPU Frequency Scaling**
  - Current governor in use
  - **Suggested governor** (highlighted in yellow)
  - EPP (Energy Performance Preference) settings
  - EPB (Energy Performance Bias) settings

- **System Statistics**
  - Total CPU usage
  - System load average
  - Average temperature across all cores
  - Turbo boost status
  - **Suggested turbo boost setting** (highlighted in yellow)

## When to Use Monitor Mode

Monitor mode is ideal in several scenarios:

<Steps>
  <Step title="Before Installing the Daemon">
    Run monitor mode to see how auto-cpufreq would optimize your system before committing to permanent changes.
  </Step>
  <Step title="Troubleshooting">
    Use monitor mode to diagnose CPU frequency scaling issues or verify that auto-cpufreq detects your hardware correctly.
  </Step>
  <Step title="Learning">
    Monitor mode helps you understand how different governors and settings affect your system under various workloads.
  </Step>
  <Step title="Comparing Settings">
    Observe system behavior with different configurations by running monitor mode with custom config files.
  </Step>
</Steps>

## Using Monitor Mode with Custom Config

You can run monitor mode with a custom configuration file to see how specific settings would behave:

```bash
sudo auto-cpufreq --monitor --config /path/to/config.conf
```

## Navigating the Monitor Interface

The monitor interface is built using a terminal UI that updates every 2 seconds:

- Press **Q** or **Ctrl+C** to quit
- Use **↑** and **↓** arrow keys to scroll
- Use **PageUp** and **PageDown** for faster scrolling

<Warning>
  If you have conflicting services like GNOME Power Profiles or TLP running, monitor mode will detect them and prompt you to continue. These services should be disabled before installing the daemon.
</Warning>
